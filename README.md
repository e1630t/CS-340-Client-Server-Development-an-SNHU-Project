# CS-340-Client-Server-Development-an-SNHU-Project
This project was developed as part of a CS 340 course assignment designed to give experience on creating reusable Python code that interacts with a NoSQL database by implementing Create, Read, Update, Delete operations.
This CRUD module then served as the foundation for developing a dashboard for user interfacing. Which gave additional experience in dashboard development and data visualization. 

**How do you write programs that are maintainable, readable, and adaptable? Especially consider your work on the CRUD Python module from Project One, which you used to connect the dashboard widgets to the database in Project Two. What were the advantages of working in this way? How else could you use this CRUD Python module in the future?**
Writing maintainable, readable, and adaptable programs requires organizing code so that each part has a clear purpose and can be reused without rewriting it. The CRUD Python modukle implements the database logic inside a single, well‑structured class with clear method names, comments, and predictable behavior, the module became easy to understand, test, and modify. This design had major advantages in Project Two. The CRUD module handled all database operations, the dashboard did not need to know anything about MongoDB becuase it called the module’s methods. This separation of concerns made the dashboard easier to build, reduced duplicated code, and allowed new features to be added without changing the underlying database logic.
This approach offered several advantages becuase it separated database logic from the dashboard interface, reduced duplicated code, simplified debugging, and made the system easier to expand. The CRUD module can also be reused in future applications with new dashboards, automation scripts or API's because it is flexible and independent of the user interface.
